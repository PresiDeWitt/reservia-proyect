## 🅰️ Vía A — Instalación local (desarrollo)

> [!abstract] Paso 1 · Clonar el repositorio
>
> ```bash
> git clone https://github.com/PresiDeWitt/reservia-proyect.git
> cd reservia-proyect
> ```

> [!abstract] Paso 2 · Configurar variables de entorno del backend
>
> Crear el archivo `backend/.env` con el siguiente contenido:
>
> ```env
> SECRET_KEY=cualquier-clave-para-desarrollo-local
> DEBUG=True
> ALLOWED_HOSTS=localhost,127.0.0.1
> CORS_ALLOWED_ORIGINS=http://localhost:5173
> OPENROUTER_API_KEY=sk-or-...   # opcional — solo para el chatbot
> ```

> [!abstract] Paso 3 · Instalar y lanzar el backend
>
> ```bash
> cd backend
> python -m venv venv
>
> # Activar entorno virtual:
> source venv/bin/activate          # macOS / Linux
> venv\Scripts\activate             # Windows
>
> pip install -r requirements.txt
> python manage.py migrate
> python manage.py seed
> python manage.py runserver
> ```
>
> ✅ Backend disponible en `http://localhost:8000`

> [!abstract] Paso 4 · Instalar y lanzar el frontend
>
> En una **nueva terminal**:
>
> ```bash
> cd frontend
> npm install
> npm run dev
> ```
>
> ✅ Frontend disponible en `http://localhost:5173`

> [!success] Verificación local
>
> | Check | URL | Esperado |
> |---|---|---|
> | Frontend carga | `http://localhost:5173` | Página con 6 restaurantes |
> | API responde | `http://localhost:8000/api/restaurants/` | JSON con restaurantes |
> | Admin Django | `http://localhost:8000/admin/` | Panel de login |

## 🅱️ Vía B — Docker (recomendado para desarrollo rápido)

> [!abstract] Paso 1 · Clonar el repositorio
>
> ```bash
> git clone https://github.com/PresiDeWitt/reservia-proyect.git
> cd reservia-proyect
> ```

> [!abstract] Paso 2 · Crear archivo `.env` en la raíz
>
> ```env
> OPENROUTER_API_KEY=sk-or-...   # opcional
> DEBUG=True
> SECRET_KEY=dev-docker-key
> ```

> [!abstract] Paso 3 · Levantar el stack completo
>
> ```bash
> docker compose up --build
> ```
>
> Docker se encarga de todo automáticamente:
>
> ```
> ① Build frontend (Node.js + Vite)         ✅
> ② Build backend (Python + pip install)    ✅
> ③ Integrar estáticos                      ✅
> ④ Aplicar migraciones                     ✅
> ⑤ Ejecutar seed (6 restaurantes)          ✅
> ⑥ Arrancar Gunicorn (3 workers)           ✅
> ```
>
> ✅ App disponible en `http://localhost`

> [!tip] Comandos Docker útiles
>
> ```bash
> docker compose up -d          # lanzar en segundo plano
> docker compose logs -f        # ver logs en tiempo real
> docker compose down           # detener (datos persistidos)
> docker compose down -v        # detener + borrar base de datos
> ```

## 🅲️ Vía C — Producción en Railway

> [!abstract] Paso 1 · Preparar el repositorio en GitHub
>
> El repositorio debe estar en GitHub con la rama `main` lista.
> El `Dockerfile` en la raíz es todo lo que Railway necesita.

> [!abstract] Paso 2 · Crear proyecto en Railway
>
> 1. Accede a [railway.app](https://railway.app) e inicia sesión
> 2. **New Project** → **Deploy from GitHub repo**
> 3. Conecta tu cuenta GitHub y selecciona el repositorio `reservia-proyect`
> 4. Railway detecta el `Dockerfile` automáticamente

> [!abstract] Paso 3 · Añadir plugin de base de datos
>
> En el panel de Railway:
> **New** → **Database** → **PostgreSQL**
>
> Railway inyecta `DATABASE_URL` automáticamente en el servicio backend.

> [!abstract] Paso 4 · Configurar variables de entorno
>
> En Railway → Service → **Variables**, añadir:
>
> | Variable | Valor |
> |---|---|
> | `SECRET_KEY` | clave aleatoria larga y segura |
> | `DEBUG` | `False` |
> | `ALLOWED_HOSTS` | `*.railway.app,tu-dominio.railway.app` |
> | `CORS_ALLOWED_ORIGINS` | `https://tu-dominio.railway.app` |
> | `OPENROUTER_API_KEY` | `sk-or-...` |
>
> ⚠️ `DATABASE_URL` la genera Railway automáticamente — no la toques.

> [!abstract] Paso 5 · Desplegar
>
> Haz push a `main`:
> ```bash
> git push origin main
> ```
>
> Railway detecta el push y ejecuta el pipeline automáticamente:
>
> ```
> Build Docker multi-stage   ✅
> Migraciones                ✅
> Seed de datos              ✅
> Gunicorn arranca           ✅
> SSL automático activo      ✅
> ```

> [!success] Verificación en producción
>
> | Check | Cómo verificar |
> |---|---|
> | App accesible | `https://tu-dominio.railway.app` en el navegador |
> | SSL activo | Candado verde en el navegador |
> | Restaurantes cargando | Página principal con 6 tarjetas |
> | Logs limpios | Railway → Deployments → ver logs |

## 🐛 Solución de problemas comunes

> [!bug] Error: `ModuleNotFoundError`
> El entorno virtual no está activado. Ejecuta `source venv/bin/activate` (Linux/Mac) o `venv\Scripts\activate` (Windows) antes de cualquier comando Python.

> [!bug] Error: CORS bloqueado en el navegador
> Verifica que `CORS_ALLOWED_ORIGINS` en `.env` coincide exactamente con la URL del frontend (sin barra final).

> [!bug] El chatbot responde "AI service not configured"
> Falta la variable `OPENROUTER_API_KEY` en el `.env`. El resto de la app funciona con normalidad sin ella.

> [!bug] Docker: puerto 80 ocupado
> Otro proceso usa el puerto 80. Modifica el `docker-compose.yml` cambiando `"80:80"` por `"8080:80"` y accede en `http://localhost:8080`.

> [!bug] Railway: deploy falla en el build
> Revisa los logs del build en Railway → Deployments. El error más común es una variable de entorno faltante.
